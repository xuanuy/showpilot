# ShowPilot — Devpost demo video (target ≤ 4:30)

Each section = one screen recording + one narration wav (`demo/vo/NN.wav`,
generated by `make_vo.sh`). Record the screen segments listed below, drop them
in `demo/rec/NN.mp4` (any length ≥ the wav), then run `build_demo.sh`.

---

## 01 · Hook (≈20s) — REC: montage of 3-4 finished reels playing (phone or FB)

> Short-form channels don't die from bad content. They die from missed days.
> A channel needs a new episode every single day — and no human team sustains
> that. So we built the team into one agent. This is ShowPilot: an autonomous
> AI showrunner.

## 02 · What it is (≈25s) — REC: README architecture diagram, slow scroll

> Give ShowPilot a one-line topic, and it does everything a studio does.
> Qwen writes the script. Qwen breaks it into a storyboard. Wan renders every
> shot. The agent voices it, edits it, captions it, publishes it — and then
> grows the audience. Not one video on demand: a fleet of channels, on a
> daily schedule.

## 03 · Live build (≈70s) — REC: terminal `python3 pipeline.py build donghua-realm the-sword-that-refused.md` (speed up waits 4-8x)

> Here it is live. The input is this topic file — a title and a theme. That's
> all a human writes. ShowPilot asks Qwen for a hundred-thirty-word script
> with a hook and a cliffhanger question. Then Qwen becomes the storyboard
> artist: each scene gets a shot prompt, and the style descriptor is repeated
> in every prompt, so characters and palette stay consistent across shots.
> Each prompt goes to Wan as an async task — vertical, nine-sixteen. The
> agent polls, downloads every shot, and caches it, so a retry never spends
> credits twice. Piper narrates. FFmpeg fits each shot to the narration,
> concatenates, burns the captions, and muxes the voice. One command in —
> one finished episode out.

## 04 · Publish + grow (≈40s) — REC: `pipeline.py post ...`, then the reel live on the FB page, first comment + story visible

> Publishing is part of the job, so the agent does that too — straight to the
> page through the Graph API. And here's the showrunner detail: it posts the
> script's closing question as the first comment, because comments are what
> the algorithm rewards, and re-shares the reel as a twenty-four-hour story.
> Every episode ships with its own growth plan.

## 05 · The fleet (≈45s) — REC: dashboard (token health, metrics), then FB pages "Donghua Decoded" / "Donghua Realm" scrolling real posted reels

> This isn't a prototype. These channels have been publishing real reels every
> day since June — unattended. Each page is a channel with its own voice, its
> own topic queue, its own credentials. A daily cron posts one new episode per
> channel, and a failed topic is logged and skipped, never fatal. The
> dashboard tracks token health, followers, and per-episode views, likes and
> comments — from a phone.

## 06 · Architecture + deploy (≈30s) — REC: code tree / lib files, then Alibaba Cloud ECS console with the instance

> The whole agent is dependency-free Python — the Qwen and Wan clients are
> standard library only. It runs on an Alibaba Cloud ECS instance in
> Singapore, same region as the Qwen endpoint, with systemd keeping the
> dashboard and the daily schedule alive.

## 07 · Close (≈15s) — REC: best generated shot full-screen, title card "ShowPilot" + repo URL

> An agent that ships daily beats a better agent that ships once. ShowPilot —
> the showrunner that never misses a day. Built on Qwen Cloud.
